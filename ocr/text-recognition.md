# Распознавание текста

> Вторая стадия OCR-пайплайна: по вырезанной области изображения с текстом предсказать последовательность символов. Главная сложность — выровнять визуальную последовательность с текстовой без посимвольных аннотаций.

## Tesseract

**Tesseract** — старейший открытый OCR-движок (HP → Google). Версии 1–3 использовали классические методы компьютерного зрения: связные компоненты, классификаторы символов. Начиная с **Tesseract 4** движок перешёл на **LSTM**: скользящее окно над строкой изображения, двунаправленный LSTM, CTC-декодер.

Сильные стороны:
- Работает «из коробки» без GPU.
- Хорошо справляется со структурированным текстом (документы, книги, DPI ≥ 300).

Ограничения:
- Чувствителен к качеству предобработки.
- Плохо справляется с «scene text»: рукописью, произвольными шрифтами, сложным фоном.

## CRNN — Convolutional Recurrent Neural Network

**CRNN (Shi et al., 2016)** — стандартная архитектура распознавания текста в эпоху до трансформеров.

Три компонента:

```
[Вырезанная строка изображения]
         ↓
  CNN (ResNet / VGG)        ← извлечение карт признаков
         ↓
  Bi-LSTM × 2               ← моделирование последовательности
         ↓
  CTC Decoder               → строка символов
```

1. **CNN** сжимает высоту изображения до 1 пикселя, производя последовательность векторов по ширине — каждый вектор описывает «колонку» исходного изображения.
2. **Bi-LSTM** улавливает зависимости между соседними колонками (контекст символа).
3. **CTC-декодер** переводит последовательность вероятностей в строку.

## CTC — Connectionist Temporal Classification

**CTC** — функция потерь и алгоритм декодирования, решающий ключевую проблему OCR: как обучить модель без разметки позиций каждого символа.

**Проблема:** изображение «кота» имеет, скажем, 24 колонки, а в слове «кот» — 3 символа. Как выровнять?

**Решение CTC:**
- Модель предсказывает для каждой временно́й точки (колонки) вероятность символа из расширенного алфавита + специальный токен blank `ε`.
- CTC суммирует вероятности всех допустимых выравниваний, которые сворачиваются в нужную строку: «к-к-о-ε-т» → «кот».
- Функция потерь: $\mathcal{L} = -\ln P(y \mid x)$, где $P(y \mid x)$ — суммарная вероятность всех путей, порождающих метку $y$.

**Допущение CTC:** монотонность — символы идут слева направо, без пропусков назад. Это ломается для арабского текста, curved text, вертикальных надписей.

## Attention-based Seq2Seq

Для нестандартных случаев (изогнутый текст, нелатинские скрипты, сложные шрифты) используют **encoder–decoder с механизмом внимания**:

1. CNN-энкодер извлекает карту признаков.
2. LSTM или трансформерный декодер генерирует символы автогрессивно.
3. На каждом шаге генерации **cross-attention** позволяет декодеру «смотреть» на нужную часть изображения.

Подробнее о cross-attention: [[neural-networks/transformer/attention-mechanism|Механизм внимания: Q, K, V]].

Это снимает ограничение монотонности CTC, но требует больше данных и медленнее на инференсе.

## TrOCR и трансформерные подходы

**TrOCR (Microsoft, 2021)** — полностью трансформерная архитектура:
- **ViT-энкодер** разбивает изображение строки на патчи и кодирует их.
- **Трансформерный декодер** генерирует текст токен за токеном через cross-attention к патчам.

Обучается на синтетических данных + дообучается на реальных. Превосходит CRNN на рукописном тексте и исторических документах.

## Сравнение методов распознавания

| Метод | Декодирование | Сильные стороны | Ограничения |
|---|---|---|---|
| Tesseract 4 | CTC (LSTM) | Простота, CPU | Качество на scene text |
| CRNN + CTC | CTC | Нет посимвольной разметки | Монотонность |
| Seq2Seq + Attention | Авторегрессия | Curved, произвольный текст | Медленнее |
| TrOCR | Авторегрессия (трансформер) | Рукопись, исторические документы | Тяжёлая модель |
