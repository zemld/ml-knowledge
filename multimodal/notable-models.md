# Известные мультимодальные модели

> Краткий разбор ключевых мультимодальных систем, каждая из которых ввела архитектурный приём, ставший индустриальным стандартом.

## CLIP (OpenAI, 2021)

**Задача:** alignment текста и изображений в общем векторном пространстве.

**Архитектура:**
- Два отдельных энкодера: визуальный (ViT или ResNet) и текстовый (GPT-2-подобный трансформер).
- Оба энкодера проецируют входы в 512-мерные эмбеддинги.

**Обучение — контрастивное (InfoNCE loss):**
- Батч из $N$ пар (изображение, текст).
- Максимизировать косинусную близость $N$ правильных пар.
- Минимизировать близость $N^2 - N$ неправильных пар.
- $$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^N \log \frac{\exp(\text{sim}(v_i, t_i)/\tau)}{\sum_{j=1}^N \exp(\text{sim}(v_i, t_j)/\tau)}$$

**Ключевое свойство — zero-shot transfer:** чтобы классифицировать изображения по новым категориям, достаточно составить текстовые шаблоны («a photo of a {class}») — без дополнительного обучения.

**Влияние:** стал backbone для DALL-E, Flamingo, Stable Diffusion и большинства современных мультимодальных моделей.

## Flamingo (DeepMind, 2022)

**Задача:** few-shot visual question answering и визуально-языковой диалог.

**Архитектура:**
- Замороженный визуальный энкодер (CLIP).
- Замороженная языковая модель (Chinchilla).
- Обучаемые **gated cross-attention** слои, вставленные между слоями LM.

**Как работает gated cross-attention:**
1. Визуальные признаки проходят через **Perceiver Resampler** — сжимаются до фиксированного числа токенов.
2. Эти токены становятся K и V в cross-attention; текстовые скрытые состояния — Q.
3. Выход cross-attention умножается на обучаемый скаляр $\alpha = \tanh(\text{gate})$, начально равный 0.
4. Это позволяет плавно «открывать» визуальный канал, не разрушая предобученный LM.

**Значение:** «GPT-3 момент» для мультимодального AI — показал, что можно не переобучать огромные LM, а добавить к ним визуальное восприятие через небольшие адаптеры.

## DALL-E / DALL-E 2 / DALL-E 3 (OpenAI, 2021–2023)

**Задача:** генерация изображений по текстовому описанию.

- **DALL-E 1:** трансформер, обученный предсказывать токены изображения (VQ-VAE) по токенам текста.
- **DALL-E 2:** использует CLIP-эмбеддинги текста как условие для диффузионной модели, которая генерирует изображение. Связь: [[neural-networks/diffusion-models/overview|Диффузионные модели]].
- **DALL-E 3:** интегрирован в ChatGPT; использует значительно улучшенные подписи к обучающим изображениям для точного следования инструкциям.

## GPT-4V / GPT-4o (OpenAI, 2023–2024)

**Задача:** универсальный мультимодальный ассистент (текст + изображения, GPT-4o также и аудио).

- Детали архитектуры не раскрыты, но визуальный энкодер (предположительно ViT) и языковая модель интегрированы глубоко.
- Умеет: описывать изображения, читать текст с фото (OCR-like), решать задачи по диаграммам, программировать по скриншотам.
- GPT-4o обрабатывает аудио нативно — без промежуточного ASR, что сохраняет интонационную информацию.

## Whisper + LLM (пайплайн)

**Задача:** понимание речевого контента на уровне смысла.

- **Whisper** (OpenAI) — ASR-модель: энкодер (аудио) + декодер (текст). Подробнее про ASR: [[asr/overview|Распознавание речи]].
- Транскрипция из Whisper подаётся в LLM (GPT, Llama) для суммаризации, ответов на вопросы, извлечения фактов.
- Это **late fusion на уровне пайплайна**: модальности не объединяются внутри сети, а последовательно обрабатываются специализированными системами.

## BLIP / BLIP-2 (Salesforce, 2022–2023)

**Задача:** единая модель для понимания и генерации (VQA, retrieval, captioning).

- **BLIP** вводит **MED** — архитектуру, которая работает как unimodal encoder, image-grounded text encoder или image-grounded text decoder в зависимости от задачи. Обучается одновременно на трёх целях: ITC (alignment), ITM (matching), LM (генерация подписей).
- **BLIP-2** добавляет **Q-Former** — лёгкий 12-слойный трансформер с обучаемыми query vectors, который извлекает релевантные визуальные признаки и подаёт их замороженному LLM как мягкие промпты. Всё дообучение сосредоточено в Q-Former; и визуальный энкодер, и LLM заморожены.

Подробный разбор: [[multimodal/blip|BLIP и BLIP-2]].

## Сравнительная таблица

| Модель | Модальности | Fusion | Ключевой приём |
|---|---|---|---|
| CLIP | Текст + Изображение | Shared encoder (contrastive) | Alignment в общем пространстве |
| Flamingo | Текст + Изображение | Intermediate (cross-attention) | Gated cross-attention к frozen LM |
| DALL-E 2 | Текст → Изображение | Условная диффузия | CLIP-эмбеддинг как условие |
| GPT-4V | Текст + Изображение | Deep integration | Нативная обработка в одной модели |
| BLIP / BLIP-2 | Текст + Изображение | Intermediate (Q-Former) | MED + Q-Former → soft prompts к frozen LLM |
| Whisper+LLM | Аудио + Текст | Late (pipeline) | Специализированные модели поэтапно |
