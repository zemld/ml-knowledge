# Метрики качества ASR

> Числовые меры, сравнивающие гипотезу ASR-системы с эталонной транскрипцией; основаны на расстоянии редактирования (Левенштейна) на уровне слов или символов.

## Общие обозначения

Все метрики строятся из четырёх базовых величин, получаемых выравниванием гипотезы (H) и эталона (R):

| Обозначение | Смысл |
|---|---|
| $S$ | Substitutions — слово заменено другим |
| $D$ | Deletions — слово из эталона пропущено |
| $I$ | Insertions — лишнее слово добавлено |
| $H$ | Hits — слово распознано верно |
| $N$ | Всего слов в эталоне: $N = H + S + D$ |

Выравнивание находится динамическим программированием — алгоритм заполняет матрицу редакционных расстояний и прослеживает путь с минимальной стоимостью.

## WER — Word Error Rate

Главная метрика ASR: доля «испорченных» позиций относительно длины эталона.

$$\text{WER} = \frac{S + D + I}{N}$$

- Диапазон: $[0, +\infty)$ — может превышать 1, если вставок больше, чем слов в эталоне.
- Человеческий уровень транскрипции: $\text{WER} \approx 5\%$.
- Не имеет верхней границы, поэтому WER сравнивает системы относительно друг друга, а не абсолютно.

## CER — Character Error Rate

Аналог WER на уровне символов. Те же $S$, $D$, $I$, но единица — символ, а не слово:

$$\text{CER} = \frac{S_c + D_c + I_c}{N_c}$$

где $N_c$ — число символов в эталоне.

CER предпочтительнее WER для языков без пробельного разделения слов (китайский, японский) и задач, где точность посимвольного воспроизведения важна (имена собственные, коды).

## MER — Match Error Rate

Вероятность того, что конкретное совпадение между гипотезой и эталоном окажется ошибочным. В отличие от WER нормируется на суммарное число позиций (включая правильные):

$$\text{MER} = \frac{S + D + I}{S + D + I + H}$$

- Диапазон: $[0, 1]$ — всегда ограничена сверху, в отличие от WER.
- При одинаковых ошибках: $\text{MER} \le \text{WER}$.

## WIP и WIL — Word Information Preserved / Lost

Информационно-теоретические метрики, оценивающие долю сохранённой смысловой информации.

**WIP** (Word Information Preserved):

$$\text{WIP} = \frac{H^2}{(H + S + D)(H + S + I)}$$

- Диапазон: $[0, 1]$; чем выше — тем лучше.
- При идеальном распознавании ($S = D = I = 0$): $\text{WIP} = 1$.

**WIL** (Word Information Lost) — дополнение к WIP:

$$\text{WIL} = 1 - \text{WIP}$$

- Диапазон: $[0, 1]$; чем ниже — тем лучше.

WIL решает проблему WER, у которого нет верхней границы: WIL всегда нормирован и ближе к тому, сколько смысла теряется при ошибках.

## Сводная таблица

| Метрика | Формула | Диапазон | Лучше когда |
|---|---|---|---|
| WER | $(S+D+I)/N$ | $[0, +\infty)$ | ниже |
| CER | $(S_c+D_c+I_c)/N_c$ | $[0, +\infty)$ | ниже |
| MER | $(S+D+I)/(S+D+I+H)$ | $[0, 1]$ | ниже |
| WIP | $H^2 / ((H+S+D)(H+S+I))$ | $[0, 1]$ | выше |
| WIL | $1 - \text{WIP}$ | $[0, 1]$ | ниже |

## Практические нюансы

**Нормализация текста перед вычислением** — обязательный шаг:
- Привести к нижнему регистру
- Удалить знаки препинания
- Развернуть сокращения и числительные в единый формат

Без нормализации метрики штрафуют за формальные различия («Hello» ≠ «hello»), не связанные с качеством распознавания.

**Case sensitivity:** по умолчанию метрики регистронезависимы — ASR не обязан воспроизводить заглавные буквы.

**Пунктуация:** обычно исключается из оценки, поскольку большинство ASR-систем не предсказывает её.

**WER > 100%:** возможно при большом числе вставок; не является ошибкой вычисления.
