# node2vec

> Алгоритм обучения векторных представлений вершин графа: генерирует «прогулки» по графу и обучает Skip-Gram модель на полученных последовательностях — так же, как word2vec обучается на текстах.

## Зачем нужны эмбеддинги вершин

Алгоритмы машинного обучения ожидают числовые векторы. Граф сам по себе — это набор узлов и рёбер, а не числа. Задача эмбеддинга — найти для каждой вершины $v$ вектор $\mathbf{z}_v \in \mathbb{R}^d$, который кодирует её место в структуре графа: похожие вершины (по роли или окружению) должны получить близкие векторы.

Эти векторы можно затем подать на вход классификатору или использовать для предсказания рёбер.

## Ключевая идея: граф → текст → word2vec

node2vec работает в два шага:

1. **Генерация случайных блужданий.** Для каждой вершины графа запускается $r$ случайных блужданий длины $l$. Каждое блуждание — последовательность вершин, как предложение из «слов».
2. **Обучение Skip-Gram.** Полученные последовательности подаются в Skip-Gram (word2vec с негативной выборкой). Модель учится предсказывать контекстные вершины по центральной — в процессе формируются эмбеддинги.

Аналогия: вершина = слово, блуждание = предложение, эмбеддинг вершины = эмбеддинг слова.

## Смещённое блуждание второго порядка

Обычное случайное блуждание (DeepWalk) выбирает следующую вершину равновероятно среди соседей. node2vec делает блуждание **смещённым** и **второго порядка**: переход из $v$ в $x$ зависит не только от текущей вершины $v$, но и от предыдущей вершины $t$.

Вероятность перехода из $v$ в $x$ пропорциональна:

$$\pi_{vx} = \alpha_{pq}(t, x) \cdot w_{vx}$$

где $w_{vx}$ — вес ребра, а $\alpha_{pq}$ — коэффициент смещения:

$$\alpha_{pq}(t, x) = \begin{cases} \frac{1}{p} & \text{если } d_{tx} = 0 \quad \text{(возврат к } t\text{)} \\ 1 & \text{если } d_{tx} = 1 \quad \text{(сосед, смежный с } t\text{)} \\ \frac{1}{q} & \text{если } d_{tx} = 2 \quad \text{(дальше от } t\text{)} \end{cases}$$

Здесь $d_{tx}$ — кратчайшее расстояние от предыдущей вершины $t$ до кандидата $x$.

## Параметры p и q

| Параметр | Название | Высокое значение | Низкое значение |
|---|---|---|---|
| **p** | return parameter | Меньше возвратов к $t$, больше исследования | Чаще возвращается к $t$ |
| **q** | in-out parameter | Держится рядом с $t$ — BFS-поведение | Уходит дальше от $t$ — DFS-поведение |

### BFS vs DFS и что они кодируют

- **Малое q (DFS):** блуждание уходит в глубину — охватывает удалённые вершины, улавливает **принадлежность к сообществам** (гомофилия: узлы одного кластера получают похожие эмбеддинги).
- **Большое q (BFS):** блуждание крутится вблизи стартовой вершины — улавливает **структурную роль** узла (два «хаба» в разных частях графа получат похожие эмбеддинги, даже не будучи соединены).

Практическая отправная точка: $p=1,\; q=0.5$.

## Пошаговый алгоритм

1. Предвычислить вероятности переходов $\pi_{vx}$ для всех рёбер (один раз).
2. Для каждой вершины сгенерировать $r$ блужданий длины $l$ с учётом смещений.
3. Собрать все блуждания в корпус последовательностей.
4. Обучить Skip-Gram с негативной выборкой (SGNS) на этом корпусе.
5. Использовать обученные эмбеддинги как признаки для downstream задач.

Благодаря предвычислению вероятностей и alias-выборке каждый шаг блуждания выполняется за $O(1)$.

## Применение эмбеддингов

Полученные векторы $\mathbf{z}_v$ используют как входные признаки для:
- **Классификации вершин** — подать $\mathbf{z}_v$ в логистическую регрессию или MLP.
- **Предсказания рёбер** — взять попарную функцию (произведение Адамара, конкатенацию) от $\mathbf{z}_u$ и $\mathbf{z}_v$.
- **Кластеризации** — применить k-means к пространству эмбеддингов.

## Ограничения

- **Трансдуктивность:** новая вершина, появившаяся после обучения, не имеет эмбеддинга — нужно переобучение. Для графов с постоянным притоком новых узлов лучше подходят индуктивные GNN (например, GraphSAGE).
- **Нет признаков вершин:** node2vec работает только с топологией. Если у узлов есть атрибуты, GNN использует их напрямую, а node2vec — нет.
- **Нет обучающего сигнала:** метод несупервайзед; на задачах с метками GNN обычно превосходит node2vec.
