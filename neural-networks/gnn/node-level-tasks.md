# Node-level задачи в GNN

> Задачи, где модель предсказывает метку или числовое свойство для каждой отдельной вершины графа.

## Суть задачи

Дан граф, часть вершин размечена. Нужно предсказать метки для оставшихся вершин, опираясь на признаки вершин и топологию соединений. Это обучение с частичной разметкой (semi-supervised): граф один, данных о соседях много, меток мало.

## Пример 1 — Обнаружение мошенничества

**Контекст:** транзакционная сеть банка или платёжной системы.

| Элемент | Что это |
|---|---|
| Вершина | Аккаунт (пользователь, карта, устройство) |
| Ребро | Транзакция между двумя аккаунтами |
| Признаки вершины | Сумма транзакций, частота, история |
| Выходной результат | Метка вершины: `мошеннический` / `легитимный` |

Мошеннические аккаунты образуют характерные паттерны в сети (звёзды, кольца), которые GNN улавливает через агрегацию соседей.

## Пример 2 — Классификация белков

**Контекст:** сеть белок-белкового взаимодействия (PPI-сеть).

| Элемент | Что это |
|---|---|
| Вершина | Белок |
| Ребро | Физическое взаимодействие двух белков в клетке |
| Признаки вершины | Последовательность аминокислот, структурные дескрипторы |
| Выходной результат | Функциональная роль белка (транспорт, катализ, сигнализация, …) |

Один белок может иметь несколько функций — задача становится многоклассовой.
