# Edge-level задачи в GNN

> Задачи, где модель предсказывает свойства существующих рёбер или определяет, должно ли между двумя вершинами появиться новое ребро (link prediction).

## Суть задачи

Модель получает эмбеддинги двух узлов и с помощью функции скоринга (dot product, MLP и т. д.) предсказывает:
- **наличие ребра** — существует ли связь (бинарная классификация),
- **тип ребра** — какой вид отношения (многоклассовая задача),
- **вес ребра** — насколько сильна связь (регрессия).

## Пример 1 — Рекомендательная система

**Контекст:** двудольный граф пользователей и товаров (или фильмов, музыки).

| Элемент | Что это |
|---|---|
| Вершина | Пользователь или товар |
| Ребро | Факт взаимодействия: покупка, просмотр, оценка |
| Признаки вершины | Профиль пользователя, характеристики товара |
| Выходной результат | Вероятность появления ребра «пользователь — товар» (=будет ли рекомендовано) |

Netflix, Spotify, Amazon используют вариации этого подхода для рекомендаций.

## Пример 2 — Предсказание взаимодействия лекарств

**Контекст:** граф лекарственных препаратов.

| Элемент | Что это |
|---|---|
| Вершина | Лекарственный препарат |
| Ребро | Известное взаимодействие (нежелательный эффект при совместном приёме) |
| Признаки вершины | Молекулярный граф препарата (SMILES), фармакологические свойства |
| Выходной результат | Тип нежелательного взаимодействия или вероятность его возникновения |

Помогает на этапе разработки лекарств выявлять опасные комбинации до клинических испытаний.
