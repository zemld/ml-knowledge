# Архитектура GAN

> GAN состоит из двух нейросетей — Генератора и Дискриминатора — которые обрабатывают данные в противоположных направлениях: G создаёт данные из шума, D оценивает их достоверность.

## Поток данных

```
Случайный шум z
      │
      ▼
┌─────────────┐
│  Генератор  │  →  Сгенерированные данные (fake)
└─────────────┘                │
                               ▼
                    ┌─────────────────────┐
Реальные данные  →  │  Дискриминатор      │  →  P(реальное) ∈ [0, 1]
                    └─────────────────────┘
```

D получает и реальные, и сгенерированные данные, и должен решить: откуда они.

## Генератор (Generator, G)

**Задача:** превратить случайный шум в данные, неотличимые от реальных.

**Вход:** вектор `z` — набор случайных чисел из нормального или равномерного распределения. Этот вектор называют **latent vector** (скрытый вектор).

**Выход:** данные нужного формата (изображение, аудио, текст).

**Архитектура для изображений:**
- Полносвязный слой: растягивает `z` в нужный размер.
- Транспонированные свёрточные слои (deconvolution / upsampling + conv): постепенно увеличивают разрешение от маленького к большому.
- Нормализация (Batch Normalization): стабилизирует обучение.
- Активация ReLU на промежуточных слоях, Tanh на выходе (масштабирует пиксели в [-1, 1]).

**Что такое Latent Space?**

Всё пространство возможных входных векторов `z` — это **latent space** (скрытое пространство). Каждая точка в нём соответствует какому-то сгенерированному объекту. Соседние точки дают похожие результаты. Именно в latent space «хранятся» концепции: форма лица, стиль рисунка, тембр голоса.

## Дискриминатор (Discriminator, D)

**Задача:** отличить реальные данные от сгенерированных.

**Вход:** данные (может быть как реальными, так и фейком от G).

**Выход:** число от 0 до 1. Близко к 1 = скорее всего реальное. Близко к 0 = скорее всего подделка.

**Архитектура для изображений:**
- Свёрточные слои (Convolution): постепенно уменьшают размер, извлекая признаки.
- Нормализация (Batch/Layer Normalization).
- Активация LeakyReLU (лучше, чем ReLU, — не «умирает» при отрицательных значениях).
- Полносвязный слой + Sigmoid: выдаёт финальную вероятность.

**Дискриминатор как адаптивная функция потерь**

D — не просто классификатор. Он сам учится тому, что значит «реалистично». Это делает его намного гибче, чем фиксированная метрика вроде pixel-wise MSE.

## Сравнение G и D

| Характеристика | Генератор (G) | Дискриминатор (D) |
|---|---|---|
| Вход | Случайный шум `z` | Данные (real или fake) |
| Выход | Синтетические данные | Вероятность [0, 1] |
| Цель | Создать убедительный фейк | Разоблачить фейк |
| Направление | Noise → Data | Data → Score |
| Типичный финальный слой | Tanh | Sigmoid |
