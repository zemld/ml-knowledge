# Real-NVP — Coupling Layer

> Real-valued Non-Volume Preserving (Real-NVP) — архитектура нормализующего потока, предложенная Dinh et al. (2016). Ключевая идея: **coupling layer** (слой связывания) — преобразование, якобиан которого является блочно-треугольным, что делает вычисление его детерминанта тривиальным.

## Прямое преобразование (x → z)

Пусть входной вектор **x** ∈ ℝᵈ разбит на две части: **x₁** (измерения 1…k) и **x₂** (измерения k+1…d).

Coupling layer задаёт:

$$\mathbf{z}_1 = \mathbf{x}_1$$

$$\mathbf{z}_2 = \mathbf{x}_2 \odot \exp\!\bigl(s(\mathbf{x}_1)\bigr) + t(\mathbf{x}_1)$$

где:
- $\odot$ — поэлементное произведение
- $s(\mathbf{x}_1)$ — **scale network**: произвольная нейросеть, принимающая **x₁** и возвращающая вектор масштабов
- $t(\mathbf{x}_1)$ — **translation network**: произвольная нейросеть, принимающая **x₁** и возвращающая вектор сдвигов

**x₁** проходит без изменений; **x₂** масштабируется и сдвигается, причём коэффициенты зависят только от **x₁**.

## Матрица Якоби и её детерминант

Матрица Якоби прямого преобразования:

$$J = \frac{\partial(\mathbf{z}_1,\, \mathbf{z}_2)}{\partial(\mathbf{x}_1,\, \mathbf{x}_2)} = \begin{pmatrix} I & 0 \\ \partial \mathbf{z}_2/\partial \mathbf{x}_1 & \operatorname{diag}\!\bigl(\exp(s(\mathbf{x}_1))\bigr) \end{pmatrix}$$

Матрица **блочно-треугольная**: верхний правый блок равен нулю, потому что **z₁** = **x₁** не зависит от **x₂** вообще.

Для блочно-треугольной матрицы детерминант равен произведению детерминантов блоков на диагонали:

$$\det J = \det(I) \cdot \det\!\bigl(\operatorname{diag}(\exp(s(\mathbf{x}_1)))\bigr) = \prod_{i=1}^{d-k} \exp\!\bigl(s_i(\mathbf{x}_1)\bigr) = \exp\!\Bigl(\sum_{i} s_i(\mathbf{x}_1)\Bigr)$$

В логарифмическом виде:

$$\log\left|\det J\right| = \sum_{i} s_i(\mathbf{x}_1)$$

Это **O(d)** вместо O(d³) — сумма выходов scale network. Сложность архитектуры сетей s и t не влияет на стоимость вычисления якобиана.

## Обратное преобразование (z → x)

$$\mathbf{x}_1 = \mathbf{z}_1$$

$$\mathbf{x}_2 = \bigl(\mathbf{z}_2 - t(\mathbf{z}_1)\bigr) \odot \exp\!\bigl(-s(\mathbf{z}_1)\bigr)$$

Для инверсии достаточно **одного прямого прохода** через s и t (с **z₁** в качестве входа). Сами сети s и t **не обязаны быть обратимыми** — их инвертировать не нужно.

## Что означает каждое обозначение

| Обозначение | Что это |
|---|---|
| **x₁**, **x₂** | Две части входного вектора **x** (разбиение произвольное) |
| **z₁**, **z₂** | Две части выходного вектора **z** (латентного представления) |
| s(**x₁**) | Scale network — задаёт логарифм масштаба для **x₂** |
| t(**x₁**) | Translation network — задаёт сдвиг для **x₂** |
| ⊙ | Поэлементное произведение (Hadamard product) |
| diag(·) | Диагональная матрица из вектора |

## Интуиция: почему якобиан треугольный

**z₁** = **x₁** — первая половина не меняется совсем, поэтому её производная по **x₂** равна нулю. Это и создаёт нулевой верхний правый блок. Производная **z₂** по **x₂** — диагональная (каждый элемент **z₂** зависит только от соответствующего элемента **x₂**). Из-за этой структуры det вычисляется просто как произведение диагональных элементов.

## Стекирование coupling layers

Один coupling layer не преобразует **x₁** — он остаётся неизменным. Чтобы обучить все измерения, coupling layers чередуют: в следующем слое роли **x₁** и **x₂** меняются местами (или используется случайная перестановка). Несколько таких слоёв образуют полноценный normalizing flow.
