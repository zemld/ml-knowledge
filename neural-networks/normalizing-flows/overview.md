# Normalizing Flows — Нормализующие потоки

> Класс генеративных моделей, в которых данные получаются из простого базового распределения через цепочку обратимых дифференцируемых преобразований. В отличие от GAN и VAE, нормализующие потоки позволяют вычислить **точное** правдоподобие данных.

## Основная идея

Пусть **z** ~ p_z(**z**) — простое базовое распределение (например, стандартный гауссиан). Нормализующий поток задаёт обратимое преобразование f: **z** → **x**, где **x** — данные (изображение, аудио и т.д.).

По [[probability-theory/change-of-variables|теореме замены переменных]] плотность данных выражается точно:

$$p_x(\mathbf{x}) = p_z(f^{-1}(\mathbf{x})) \cdot \left|\det J_{f^{-1}}(\mathbf{x})\right|$$

Или, логарифмируя (что удобнее для оптимизации):

$$\log p_x(\mathbf{x}) = \log p_z(\mathbf{z}) + \log\left|\det J_{f^{-1}}\right|$$

## Почему это ценно

| Свойство | GAN | VAE | Normalizing Flow |
|---|---|---|---|
| Точное правдоподобие | ✗ | Нижняя оценка | **✓** |
| Обратимая генерация | ✗ | ✗ | **✓** |
| Латентное пространство | Неявное | Приближённое | **Точное** |

## Главная инженерная задача

Вычисление $\det J$ для матрицы размера n×n стоит O(n³) — слишком дорого для изображений. Архитектуры нормализующих потоков специально конструируют преобразования с **треугольным якобианом**, где детерминант вычисляется за O(n).
