# Текстовая диффузия (Text Diffusion)

> Применение диффузионных моделей к генерации текста — итеративное восстановление последовательности токенов из зашумлённого или замаскированного состояния, позволяющее генерировать весь текст параллельно, а не токен за токеном.

## Проблема: текст дискретен

DDPM работает с непрерывными данными — изображениями (см. `[[neural-networks/diffusion-models/ddpm|DDPM]]`). Гауссов шум можно добавить к пикселю как к числу, и нейросеть предсказывает, сколько шума убрать.

С текстом это не работает напрямую: токен — это целое число-идентификатор из словаря (например, «кот» = 1423). Нельзя «немного зашумить» токен, сложив с гауссовым шумом.

Два основных подхода к решению:

| Подход | Идея | Пример модели |
|---|---|---|
| **Диффузия в пространстве эмбеддингов** | Токены → непрерывные векторы, шум добавляется к ним | Diffusion-LM |
| **Дискретная (маскированная) диффузия** | Forward process заменяет токены на `[MASK]`, reverse — восстанавливает | MDLM, LLaDA |

## Маскированная диффузия (Masked Diffusion)

Наиболее перспективный на 2024–2025 гг. подход. Forward process — это стохастическое **маскирование** токенов:

$$q(x_t \mid x_0): \text{каждый токен с вероятностью } \beta_t \text{ заменяется на } [\text{MASK}]$$

При $t = T$ вся последовательность полностью замаскирована. Reverse process — трансформер-декодер учится восстанавливать токены из частично замаскированной последовательности:

$$p_\theta(x_0 \mid x_t): \text{предсказать оригинальный токен для каждого } [\text{MASK}]$$

Функция потерь упрощается до **взвешенной суммы MLM-потерь** (masked language modeling) по всем временным шагам — та же кросс-энтропия, что в BERT.

**Генерация:** начинаем с полностью замаскированной последовательности фиксированной длины. На каждом шаге трансформер предсказывает все токены параллельно, наиболее уверенные из них «фиксируются», остальные остаются замаскированными. Процесс повторяется $T$ шагов.

## Ключевые модели

- **MDLM** (Sahoo et al., NeurIPS 2024) — достигает state-of-the-art perplexity среди диффузионных моделей на LM1B, конкурентен с авторегрессивными моделями.
- **LLaDA 8B** — предобученная модель масштаба 8B параметров на masked diffusion, сопоставима с авторегрессивными LLM того же размера.

## Отличия от авторегрессивного трансформера

| Характеристика | Авторегрессивный (GPT-like) | Текстовая диффузия |
|---|---|---|
| **Порядок генерации** | Слева направо, токен за токеном | Параллельно, весь текст за $T$ итераций |
| **Контекст** | Только левый (causal mask) | Двунаправленный (bidirectional attention) |
| **Уточнение** | Нельзя исправить уже выданный токен | Каждая итерация может изменить любой токен |
| **Задержка инференса** | Пропорциональна длине текста | Фиксированные $T$ шагов, независимо от длины |
| **Контроль формата** | Через промпт / принудительное декодирование | Можно «заморозить» произвольные позиции до генерации |

**Главный выигрыш диффузии:** токены на разных позициях могут уточняться одновременно с учётом правого контекста. Авторегрессивная модель, выдав слово в позиции 3, не может его изменить, даже если слово в позиции 10 делает его неуместным.

**Главный проигрыш диффузии:** при фиксированном бюджете вычислений авторегрессивные модели пока всё ещё превосходят диффузионные по качеству на большинстве benchmark'ов; диффузия требует выбора длины выходной последовательности заранее.
