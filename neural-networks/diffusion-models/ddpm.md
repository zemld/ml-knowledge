# DDPM — Denoising Diffusion Probabilistic Models

> Базовая архитектура диффузионных моделей (Ho et al., 2020). Нейросеть обучается предсказывать шум, добавленный к изображению, и использует это для пошагового восстановления изображений из чистого гауссового шума.

## Прямой процесс (forward process)

Прямой процесс — это фиксированная марковская цепь, которая постепенно добавляет гауссов шум к изображению **x₀** за T шагов:

$$q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) = \mathcal{N}\!\left(\mathbf{x}_t;\; \sqrt{1 - \beta_t}\,\mathbf{x}_{t-1},\; \beta_t\mathbf{I}\right)$$

где β₁, β₂, …, β_T — заранее заданный **noise schedule** (расписание шума), небольшие числа от ~0.0001 до ~0.02.

**Ключевое свойство**: зашумлённое изображение на шаге t можно получить напрямую из **x₀** без итерации:

$$q(\mathbf{x}_t \mid \mathbf{x}_0) = \mathcal{N}\!\left(\mathbf{x}_t;\; \sqrt{\bar\alpha_t}\,\mathbf{x}_0,\; (1 - \bar\alpha_t)\mathbf{I}\right)$$

или в форме для выборки:

$$\mathbf{x}_t = \sqrt{\bar\alpha_t}\,\mathbf{x}_0 + \sqrt{1 - \bar\alpha_t}\,\boldsymbol\varepsilon, \quad \boldsymbol\varepsilon \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$$

где $\bar\alpha_t = \prod_{s=1}^{t}(1 - \beta_s)$. При t = T значение $\bar\alpha_T \approx 0$, то есть **x_T** — почти чистый шум.

## Что предсказывает нейросеть

Нейросеть ε_θ(**x_t**, t) обучается предсказывать **шум ε**, который был добавлен к изображению:

- Вход: зашумлённое изображение **x_t** + номер шага t
- Выход: предсказанный шум ε̂ той же размерности, что и изображение

Зная предсказанный шум, можно восстановить оценку оригинала **x₀** или перейти к менее зашумлённому **x_{t-1}**.

## Функция потерь

Из вариационной нижней оценки ELBO выводится упрощённая функция потерь:

$$\mathcal{L}_{\text{simple}} = \mathbb{E}_{t,\,\mathbf{x}_0,\,\boldsymbol\varepsilon}\!\left[\,\left\|\boldsymbol\varepsilon - \boldsymbol\varepsilon_\theta(\mathbf{x}_t, t)\right\|^2\right]$$

**MSE между реальным шумом ε и предсказанным шумом ε_θ** — простейшая регрессия. Модель не видит ни оригинал, ни чистое изображение напрямую: только **x_t** и t.

## Алгоритм обучения

```
repeat:
  1. Взять x₀ из обучающей выборки
  2. Выбрать случайный шаг t ~ Uniform{1, …, T}
  3. Сгенерировать шум ε ~ N(0, I)
  4. Вычислить x_t = sqrt(ᾱ_t)·x₀ + sqrt(1-ᾱ_t)·ε
  5. Вычислить потери: ||ε - ε_θ(x_t, t)||²
  6. Обновить θ градиентным спуском
until сходится
```

На каждой итерации модель обучается «угадывать», какой шум был добавлен к конкретному зашумлённому изображению на конкретном шаге.

## Архитектура нейросети: U-Net

Стандартная архитектура ε_θ — это **U-Net** с дополнениями для работы с временным шагом.

### Структура U-Net

```
Encoder (downsampling):
  x_t → Conv → ResBlock → ResBlock → Downsample →
         Conv → ResBlock → ResBlock → Downsample → ...

Bottleneck:
  ... → ResBlock → Attention → ResBlock → ...

Decoder (upsampling):
  ... → Upsample → ResBlock → ResBlock → Conv →
        Upsample → ResBlock → ResBlock → Conv → ε̂
```

- **Skip connections**: encoder передаёт признаки напрямую в decoder на каждом уровне масштаба — как в классическом U-Net для сегментации
- **ResBlock**: остаточные блоки (residual blocks) для стабильного обучения глубоких сетей
- **Attention**: слои self-attention добавляются на средних пространственных разрешениях для захвата глобальных зависимостей

### Как подаётся временной шаг t

t кодируется в вектор с помощью **sinusoidal positional embedding** (аналогично трансформерам):

$$\text{emb}(t) = \left[\sin(t / 10000^{2i/d}),\; \cos(t / 10000^{2i/d})\right]_{i=1}^{d/2}$$

Этот вектор проецируется через линейный слой и добавляется к активациям каждого ResBlock через addition или scale-shift:

$$\text{out} = \text{GroupNorm}(h) \cdot (1 + \gamma(t)) + \delta(t)$$

где γ(t) и δ(t) — выходы линейных слоёв над embedding t.

**Итого, модель принимает на вход:**
- Зашумлённое изображение **x_t** размером H × W × C
- Временной шаг t (целое число от 1 до T) — через embedding, встроенный в каждый слой

## Генерация (sampling)

После обучения новые изображения генерируются так:

```
1. Взять x_T ~ N(0, I)  ← чистый шум
2. Для t = T, T-1, ..., 1:
   a. Предсказать шум: ε̂ = ε_θ(x_t, t)
   b. Вычислить x_{t-1} по формуле обратного процесса:
      x_{t-1} = (1/sqrt(α_t)) · (x_t - β_t/sqrt(1-ᾱ_t) · ε̂) + sigma_t · z
      где z ~ N(0,I) если t > 1, иначе z = 0
3. Вернуть x_0
```

Генерация требует T прямых проходов через сеть (T = 1000 в оригинальной статье) — отсюда медленность по сравнению с GAN.

## Что означает каждое обозначение

| Обозначение | Что это |
|---|---|
| **x₀** | Оригинальное изображение из датасета |
| **x_t** | Изображение после t шагов зашумления |
| **x_T** | Почти чистый гауссов шум |
| βt | Дисперсия добавляемого шума на шаге t (noise schedule) |
| ᾱ_t | Произведение (1−β₁)·…·(1−βt) — суммарное «выживание» сигнала |
| ε | Реальный шум, добавленный при зашумлении (таргет) |
| ε_θ | Нейросеть, предсказывающая шум |
| t | Временной шаг (от 1 до T) |
| T | Общее число шагов диффузии (обычно 1000) |
