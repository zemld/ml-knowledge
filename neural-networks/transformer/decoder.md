# Декодер Transformer: три слоя внимания

> Каждый блок декодера содержит три подслоя: маскированное самовнимание, кросс-внимание и полносвязную сеть — вместе они учат декодер генерировать токены, опираясь и на уже сгенерированный текст, и на входную последовательность.

## Структура блока декодера

```
Вход (embeddings предыдущих токенов)
        │
        ▼
┌──────────────────────────────────┐
│  1. Masked Multi-Head Attention  │  ← смотрит только на прошлые токены
└────────────────┬─────────────────┘
                 │ + Residual / LayerNorm
                 ▼
┌──────────────────────────────────┐
│  2. Cross-Attention              │  ← Q из декодера, K/V из энкодера
└────────────────┬─────────────────┘
                 │ + Residual / LayerNorm
                 ▼
┌──────────────────────────────────┐
│  3. Feed-Forward Network         │
└────────────────┬─────────────────┘
                 │ + Residual / LayerNorm
                 ▼
             Выход блока
```

Каждый подслой обёрнут в остаточное соединение и нормализацию:
$$\text{Output} = \text{LayerNorm}(x + \text{Sublayer}(x))$$

## 1. Masked Self-Attention (маскированное самовнимание)

**Что делает.** Позволяет каждому токену декодера посмотреть на все предыдущие токены в текущей генерируемой последовательности.

**Откуда Q, K, V.** Все три вектора строятся из одной и той же последовательности — выхода предыдущего блока декодера.

**Маска.** Чтобы при генерации позиции $i$ декодер не видел позиции $i+1, i+2, \ldots$, к матрице скоров прибавляется маска:

$$\text{score}_{ij} = \begin{cases} \text{score}_{ij}, & j \le i \\ -\infty, & j > i \end{cases}$$

После softmax веса для будущих позиций превращаются в 0 — они не влияют на результат.

**Зачем нужна.** Transformer обучается на полных последовательностях, обрабатывая все позиции параллельно. Маска имитирует авторегрессионную генерацию: «при предсказании токена $i$ мы знаем только токены до $i$».

## 2. Cross-Attention (перекрёстное внимание)

**Что делает.** Связывает декодер с кодированием входной последовательности: декодер «спрашивает» у энкодера, какая часть входа сейчас важна.

**Откуда Q, K, V:**
- **Q** — из выхода предыдущего подслоя декодера (masked self-attention + LayerNorm).
- **K и V** — из выхода энкодера (вычисляются один раз и используются в каждом блоке декодера).

Формула та же, что для обычного attention:
$$\text{CrossAttention}(Q_{\text{dec}},\, K_{\text{enc}},\, V_{\text{enc}}) = \text{softmax}\!\left(\frac{Q_{\text{dec}}\,K_{\text{enc}}^\top}{\sqrt{d_k}}\right)V_{\text{enc}}$$

**Зачем нужен.** Это единственное место, где информация из энкодера попадает в декодер. Q несёт контекст «что мы уже сгенерировали», а K/V несут «что есть во входе» — attention выбирает наиболее релевантные части входа для текущего шага генерации.

**Пример.** При машинном переводе «The cat sat» → «Le chat était assis» декодер при генерации «chat» построит Q, заряженный контекстом уже выданного «Le», и через cross-attention найдёт в энкодерных K/V наиболее весомое слово «cat».

## 3. Feed-Forward Network (FFN)

Применяется позиционно-независимо к каждому токену:

$$\text{FFN}(x) = \max(0,\, xW_1 + b_1)\,W_2 + b_2$$

Расширяет представление, давая сети нелинейное преобразование после этапа внимания.

## Как слои взаимодействуют

1. **Masked self-attention** строит богатое представление «что мы уже сгенерировали», позволяя каждому токену учитывать весь предыдущий контекст.
2. Это представление становится **Q для cross-attention** — поэтому запросы к энкодеру учитывают уже сгенерированный контекст.
3. Cross-attention добавляет информацию из входной последовательности, FFN финально трансформирует вектор.

## Декодер без энкодера (decoder-only модели)

В моделях типа GPT энкодера нет, поэтому cross-attention слой убирается. Каждый блок содержит только masked self-attention и FFN. Вся информация хранится в самой последовательности.
