# AI-агенты на базе LLM

> Агент — это LLM, помещённый в цикл рассуждение–действие: он получает цель, разбивает её на шаги, вызывает инструменты, запоминает результаты и повторяет до завершения задачи.

## Чем агент отличается от обычного LLM

Обычная языковая модель отвечает на один промпт и останавливается. Агент работает в **петле**:

```
Цель → [Думает → Выбирает действие → Выполняет → Наблюдает результат] → повтор → Ответ
```

Именно цикл и возможность вызывать внешние инструменты превращают пассивный чатбот в систему, способную самостоятельно решать многошаговые задачи.

## Четыре компонента агента

### LLM-«мозг» (Agent Core)

Центральная языковая модель, которая:
- интерпретирует цель и текущее состояние,
- решает, какое действие предпринять следующим,
- формулирует финальный ответ.

Качество модели задаёт потолок всего, что агент способен сделать. Слабая модель не справится со сложным планированием даже при идеальных инструментах.

### Инструменты (Tools)

Функции, которые агент может вызывать: поисковые API, калькулятор, база данных, браузер, код-интерпретатор, внешние сервисы. Без инструментов агент ограничен только параметрической памятью модели.

Каждый инструмент описывается схемой: имя, назначение, параметры. LLM читает описания и выбирает подходящий.

### Память (Memory)

| Тип | Где хранится | Назначение |
|---|---|---|
| **Краткосрочная** | контекстное окно | текущие мысли, история шагов, наблюдения |
| **Долгосрочная** | векторная база данных | знания, результаты прошлых сессий |

Контекстное окно пересобирается при каждом вызове — без долгосрочной памяти агент «забывает» всё между сессиями.

### Планировщик (Planner)

Определяет стратегию разбивки задачи на шаги. Может работать двумя способами:
- **С обратной связью** (ReAct) — шаги формируются динамически, после каждого наблюдения.
- **Plan-and-Execute** — сначала строится полный план, потом он исполняется шаг за шагом.

## Многоагентные системы

Сложные задачи распределяются между несколькими специализированными агентами: один — планировщик, другой — исследователь, третий — исполнитель. Агенты общаются через сообщения, передавая контекст и результаты.
