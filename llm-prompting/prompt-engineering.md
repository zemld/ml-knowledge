# Техники промпт-инжиниринга

> Промпт-инжиниринг — набор приёмов формулировки промпта, которые помогают модели выдавать более точные и полезные ответы без изменения весов модели.

## Zero-Shot

Промпт без примеров: только задача, никаких образцов. Модель опирается исключительно на знания, полученные при предобучении.

```
Переведи на английский: "Привет, как дела?"
```

**Когда работает хорошо:** простые, однозначные задачи, где формат ответа очевиден.
**Ограничение:** для нестандартного формата вывода или узкоспециализированных задач модель может ошибиться.

## Few-Shot

В промпт добавляются 2–5 примеров «вход → выход». Модель улавливает паттерн и воспроизводит его для нового запроса.

```
Классифицируй тональность:
"Отличный сервис!" → позитивная
"Долгая очередь."   → негативная
"Стандартный номер." → нейтральная
"Персонал был груб." → ???
```

**Когда использовать:** сложные domain-specific задачи, нужен строго определённый формат вывода (JSON, YAML), или поведение отличается от «умолчаний» модели.
**Правило:** все примеры должны иметь одинаковую структуру, иначе модель запутается.

## Chain-of-Thought (CoT)

Промпт просит модель рассуждать пошагово, прежде чем дать финальный ответ. Промежуточные шаги называются **reasoning chain** (цепочкой рассуждений).

**Zero-Shot CoT** — достаточно добавить фразу в конец промпта:
```
Реши задачу. Думай шаг за шагом.

У Маши 5 яблок. Она отдала 2 Коле и купила 3 ещё. Сколько теперь?
```

**Few-Shot CoT** — примеры содержат не только ответ, но и рассуждение:
```
Вопрос: 15 + 27 = ?
Рассуждение: 15 + 20 = 35, затем 35 + 7 = 42.
Ответ: 42

Вопрос: 38 + 46 = ?
```

**Когда помогает:** многошаговые задачи — математика, логика, планирование. Для простых задач прирост незначительный, а ответ становится длиннее.

## Сравнительная таблица

| Техника | Примеры | Рассуждение | Когда применять |
|---|---|---|---|
| Zero-Shot | нет | нет | простые, быстрые задачи |
| Few-Shot | есть | нет | сложный формат, узкая область |
| Chain-of-Thought | опц. | есть | многошаговые рассуждения |
