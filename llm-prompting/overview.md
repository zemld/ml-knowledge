# Промпты в LLM

> Промпт — это текстовый вход, который пользователь или система передаёт языковой модели; модель обрабатывает его как последовательность токенов в контекстном окне и генерирует продолжение.

## Что такое промпт

Промпт — это любой текст, подаваемый модели на вход. Модель не «понимает» задачу в человеческом смысле: она просто предсказывает наиболее вероятное продолжение данной последовательности токенов. Поэтому формулировка промпта напрямую влияет на качество ответа.

Современные LLM-API структурируют промпт через **роли** — явные метки, которые помогают модели различать, кто что сказал и в каком порядке.

## Три роли

| Роль | Кто задаёт | Назначение |
|---|---|---|
| **system** | разработчик / система | глобальные инструкции: личность модели, правила поведения, формат вывода |
| **user** | конечный пользователь | конкретный запрос или вопрос |
| **assistant** | сама модель | предыдущие ответы модели в диалоге |

**System** задаёт контекст один раз и остаётся невидимым для пользователя, но модель читает его при каждом обращении. **User** и **assistant** чередуются в многоходовых диалогах, передавая историю переписки.

Пример структуры одного API-вызова:
```
system:    "Ты — дружелюбный ассистент по математике."
user:      "Сколько будет 7 × 8?"
assistant: "56."
user:      "А в квадрате?"
```

## Контекстное окно

Всё, что видит модель при генерации одного ответа, называется **контекстным окном** — это фиксированный буфер токенов:

$$\text{context} = T_{\text{system}} + T_{\text{history}} + T_{\text{user}} + T_{\text{tools}} + T_{\text{output}}$$

- Измеряется в **токенах** (≈ 0.75 слова для английского).
- При превышении лимита старые сообщения обрезаются — обычно сначала удаляется история диалога.
- Контекст **пересобирается с нуля** при каждом API-вызове: модель не хранит состояние между вызовами.

Модель не «помнит» предыдущие разговоры — она лишь видит токены, переданные в текущем вызове.
