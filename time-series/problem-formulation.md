# Постановка задачи прогнозирования временного ряда

> Прогнозирование временного ряда — это задача предсказания будущих значений последовательности на основе её прошлых наблюдений; ключевые параметры — горизонт прогноза и размер окна контекста.

## Формальное определение

Временной ряд — это последовательность $\{y_1, y_2, \ldots, y_T\}$, где $y_t \in \mathbb{R}$ — наблюдение в момент времени $t$, а $T$ — длина ряда.

**Задача прогнозирования**: по наблюдениям $y_1, \ldots, y_t$ предсказать будущие значения $y_{t+1}, \ldots, y_{t+H}$, где $H$ — **горизонт прогноза** (forecast horizon).

Формально строится функция $f$:

$$\hat{y}_{t+1}, \ldots, \hat{y}_{t+H} = f(y_{t-p+1}, \ldots, y_t)$$

где $p$ — **размер окна** (lookback window, lag window): сколько прошлых значений подаётся на вход модели.

## Горизонт прогноза

Горизонт $H$ определяет, насколько далеко в будущее нужно смотреть:

- **$H = 1$** — одношаговое прогнозирование (one-step-ahead): предсказывается только следующий момент. Самый простой случай.
- **$H > 1$** — многошаговое прогнозирование (multi-step): предсказываются $H$ следующих значений.

Чем больше горизонт, тем сложнее задача: неопределённость нарастает, а ошибки из ближних шагов могут влиять на дальние.

## Одношаговое vs многошаговое

| | Одношаговое ($H=1$) | Многошаговое ($H>1$) |
|---|---|---|
| Сложность | Ниже | Выше |
| Ошибки | Накапливаются только если применять рекурсивно | Зависят от выбранной стратегии |
| Типичное применение | Оценка модели, онлайн-системы | Планирование, бизнес-прогнозы |

Для многошагового прогнозирования нужно выбрать **стратегию** — способ применения модели(ей) для получения $H$ прогнозов. Разные стратегии по-разному решают вопрос о накоплении ошибок и зависимости между шагами.
