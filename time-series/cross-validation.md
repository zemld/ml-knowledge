# Кросс-валидация временных рядов

> Стандартный K-fold разрушает порядок наблюдений и порождает утечку данных из будущего; специальные стратегии — Walk-forward, Blocked и Purged CV — устраняют эту проблему, всегда обучая на прошлом и проверяя на будущем.

## Почему обычный K-fold не работает

Стандартная K-fold кросс-валидация случайно перемешивает наблюдения и разбивает их на фолды. Для временных рядов это нарушает два фундаментальных допущения:

1. **Нарушение временного порядка.** Тренировочный фолд может содержать наблюдения, которые хронологически позже тестовых. Модель обучается «на будущем» относительно своей же тестовой выборки.

2. **Утечка данных (data leakage).** Наблюдения в соседних временных точках коррелированы. Попав вместе в разные фолды, они передают информацию из теста в обучение, искусственно занижая ошибку и создавая ложно-оптимистичные оценки.

3. **Нарушение допущения i.i.d.** K-fold предполагает независимые и одинаково распределённые наблюдения. Временные ряды автокоррелированы по определению — это допущение нарушено.

Признак скрытой утечки: если ошибка валидации подозрительно стабильна во всех фолдах, а на реальных данных модель резко деградирует — это сигнал, что K-fold использовался там, где не следовало.

## Walk-forward: расширяющееся окно (Expanding Window)

На каждом шаге обучение идёт на **всех** доступных прошлых данных, а тест — на следующем фиксированном отрезке:

```
Fold 1: [■■■■□□□□□□□□□□□□]   Train=4, Test=1
Fold 2: [■■■■■□□□□□□□□□□□]   Train=5, Test=1
Fold 3: [■■■■■■□□□□□□□□□□]   Train=6, Test=1
         ←train→ ←test→
```

**Плюсы:** имитирует реальный процесс обновления модели; чем дальше фолд, тем больше данных — модель становится лучше с ростом истории.

**Минусы:** дорого вычислительно — последние фолды обучаются на огромной выборке; более ранние фолды могут плохо отражать поведение модели в боевом режиме.

## Walk-forward: скользящее окно (Sliding / Rolling Window)

Обучение и тест движутся вперёд с **фиксированным** размером окна:

```
Fold 1: [■■■■□□□□□□□□□□□□]   Train=4, Test=1
Fold 2: [○■■■■□□□□□□□□□□□]   Train=4, Test=1
Fold 3: [○○■■■■□□□□□□□□□□]   Train=4, Test=1
```

Старые данные выходят из обучения по мере добавления новых.

**Плюсы:** устойчив к нестационарности — модель всегда видит «свежий» срез ряда; постоянная стоимость обучения на каждом фолде.

**Минусы:** теряются исторические паттерны; нужно подбирать размер окна.

## Blocked K-fold (Блочная кросс-валидация)

Ряд делится на $k$ последовательных блоков; тест всегда берётся из блока, который стоит **после** тренировочных блоков в этом разбиении:

```
Fold 1: [■■■■■][□□□□□][─────][─────]
Fold 2: [─────][■■■■■][□□□□□][─────]
Fold 3: [─────][─────][■■■■■][□□□□□]
```

Блоки не перемешиваются — порядок сохраняется внутри каждого фолда.

**Плюсы:** ближе к обычному K-fold по числу оценок; хорошо работает на длинных рядах с сезонностью.

**Минусы:** несколько блоков между тестом и обучением могут оказаться «выброшенными»; сложнее интерпретировать, чем Walk-forward.

## Purged Cross-Validation с эмбарго (финансовые ряды)

Введена Маркосом Лопес де Прадо (2018) для финансовых временных рядов, где метки (labels) **зависят от пути** (path-dependent): метка актива в момент $t$ определяется тем, что произошло в промежутке $[t, t + \Delta]$ — например, достигла ли цена стоп-лосса.

Такие перекрывающиеся метки порождают скрытую утечку даже в Walk-forward схемах. Решение — два дополнительных шага:

1. **Purging (очистка).** Из тренировочного фолда удаляются все наблюдения, чьи метки перекрываются по времени с метками тестового фолда.

2. **Embargo (эмбарго).** После тестового фолда добавляется буфер из $e$ наблюдений, которые также исключаются из ближайшего тренировочного фолда — они могут нести сериально зависимые признаки.

```
[■■■ TRAIN ■■■] [×× purge ××] [□ TEST □] [≈≈ embargo ≈≈] [■■ TRAIN ■■]
```

**Плюсы:** устраняет самую тонкую форму утечки в перекрывающихся временных метках.

**Минусы:** требует явного задания размера эмбарго и понимания структуры меток; специфично для финансовых задач.

## Сравнение стратегий

| Стратегия | Порядок сохранён | Утечка | Вычислительная стоимость | Область применения |
|---|---|---|---|---|
| **K-fold** | ✗ | Высокая | Низкая | i.i.d. данные |
| **Expanding Window** | ✓ | Нет | Растёт с фолдами | Стационарные ряды |
| **Sliding Window** | ✓ | Нет | Константная | Нестационарные ряды |
| **Blocked K-fold** | ✓ | Низкая | Средняя | Длинные ряды с сезонностью |
| **Purged + Embargo** | ✓ | Нет | Средняя | Финансовые ряды с path-dependent метками |
